# Growth OS — Rollout Strategy

_Prioritized sequence, written to teach the reasoning so it can be explained in
the room, not just followed._

## The principle

Build in the order that makes each next step cheaper and safer. Score before you
alert (you can't alert on a score that doesn't exist). Gate before you send
(compliance before any partner-facing message goes live). Prove value with the
highest-ROI, lowest-risk agent first, so trust is earned before spend grows.

## Do this first — Lead Scoring + Speed-to-Lead (Week 1)

**Why first:** everything downstream depends on the score, and speed-to-lead is
the single highest-ROI move in B2B sales (21× qualification lift inside 5
minutes). Speed-to-Lead is internal-facing (it texts the BDR, not a partner), so
it needs no compliance gate yet — a fast, safe first win.
**Result:** every new agency prospect is instantly ranked and the BDR is tapped
on the shoulder for the hot ones. This alone changes the day.

## Next — Compliance Guardian, in shadow mode (Week 1–2)

**Why here:** it must exist before any partner-facing sender goes live, but run
it in shadow mode against real drafts for a week first so you trust its
judgment before it can block a send. Standing it up early de-risks everything
after it.

## Then — Nurture + Call Prep (Week 2–3)

**Why:** Nurture Drip is the first partner-facing sender (now safely behind the
gate) and covers the warm middle of the funnel that otherwise goes cold. Call
Prep is internal, independent, and directly lifts close rate — a fast second win
while the nurture data accumulates.

## Then — the recovery + retention agents (Week 3–4)

No-Show Rescue, Referral Ask, Partner Retention. These protect revenue you've
already worked for — and because commission is recurring, retention compounds.

## Then — content + ops (Week 4+)

Content Idea + Hook Research feed the top of funnel; Pipeline QA, Income
Reconciliation, and Daily Briefing keep the whole system honest and visible.
Affiliate Attribution launches in shadow mode (real partner payouts).

## How to run it

Open the **AI Team** tab, copy the **master setup prompt** into a fresh Claude
chat, and let Claude walk the build one agent at a time in exactly this order —
asking what you have access to, guiding each config step, and testing before
moving on. The **Build** tab tracks the 8 phases behind it. The **Triage Strip**
tells the BDR the highest-value move on any given day once it's running.
